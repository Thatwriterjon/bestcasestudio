# Documentation: Knowledge Base as Credibility

## Why Documentation Builds Credibility

Comprehensive documentation signals:
- "We've thought about how people will use this"
- "We want customers to succeed"
- "We're a professional organization"

Missing documentation signals:
- "We didn't think about onboarding"
- "Customers should figure it out"
- "We're not organized"

## Types of Documentation

**Getting started guide:** First steps. How to sign up, initial setup, first win.

**Feature documentation:** How each feature works. Step-by-step guides.

**API documentation:** For developer products. How to integrate and use APIs.

**Common workflows:** Step-by-step guides for typical customer tasks.

**FAQ:** Answers to frequently asked questions.

**Troubleshooting:** How to fix common issues.

**Best practices:** Tips for using the product effectively.

## Documentation Quality Signals

**Strong documentation:**
- Searchable (easy to find answers)
- Well-organized (clear structure)
- Current (not outdated)
- Specific (real examples, not vague)
- Visual (screenshots, videos, diagrams)
- Accessible (works on mobile, accessible to disabled users)

**Weak documentation:**
- Not searchable
- Disorganized
- Outdated
- Vague
- Text-only
- Not mobile-friendly

## Public vs. Private Documentation

**Public documentation:**
- Customers can find answers without support
- Reduces support load
- Improves credibility (transparent)
- Helps with SEO (documentation content ranks)
- Shows you want customers to self-serve

**Private documentation:**
- Only accessible to customers
- Better for sensitive content
- Less SEO value
- Less credibility signal

**Best practice:** Make documentation public. Private docs are fine for sensitive content, but most documentation should be public.

## Documentation as Marketing

Good documentation ranks in search results.

Example: Someone searches "how to set up email automation." If your guide is the best result, you win that customer before they even know you exist.

**How documentation helps SEO:**
- Ranks for long-tail keywords
- Builds topical authority
- Generates backlinks
- Keeps users on site longer
- Signals expertise to search engines

## Keeping Documentation Current

Outdated documentation destroys credibility more than no documentation.

"This feature was removed in version 2.0" looks bad. Update it.

**How to keep docs current:**
- Review quarterly
- Update when features change
- Mark outdated content clearly
- Archive old versions
- Have process for documentation updates

## Video Documentation

Combining video with text documentation improves learning.

**Video works well for:**
- Workflows (step-by-step)
- Feature walkthroughs
- UI navigation
- Complex processes

**Text documentation works well for:**
- Reference material
- Searchability
- Accessibility
- Detailed technical information

**Best practice:** Video + text. Video shows, text documents.

## Knowledge Base Platforms

**Self-hosted:**
- Full control
- High effort to maintain
- Takes technical resources
- SEO benefit

**Third-party platforms:**
- Zendesk, Intercom, Notion, etc.
- Easier to manage
- Lower technical overhead
- Less control

**Recommendation:** Start with third-party (easier to build). Graduate to self-hosted as you scale.

## Documentation Credibility Signals

**Strong documentation:**
- Comprehensive (covers all features)
- Well-organized and searchable
- Current (not outdated)
- Public and SEO-friendly
- Includes videos and visuals
- Regularly updated

**Weak documentation:**
- Sparse (missing features)
- Disorganized, hard to search
- Outdated (references removed features)
- Private or hidden
- Text-only
- Never updated

## Key Takeaway

Documentation credibility proves that you care about customer success.

Create comprehensive, searchable, current documentation. Make it public. Update it regularly.

That's how SaaS companies build credibility through helpful resources.
